# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha:spark:flush-resets-state]** `SugarCraft\Ansi\Parser\Parser::flush()` resets the parser to `State::Ground`. Any end-of-stream decision that depends on the parser having ended mid-sequence (e.g. `currentState() === State::Escape` to detect a trailing bare ESC) MUST capture the state **before** calling `flush()`. `StreamingInspector::finish()` originally checked `currentState()` *after* `flush()`, so the branch was permanently dead and a lone trailing ESC (`"\x1b"` with no following byte) was silently dropped. Capture `$stateBeforeFlush = $parser->currentState();` first, then flush, then decide. `AnsiHandler::parse()` already did this correctly — `finish()` did not.
- **[gotcha:spark:foreign-visibility]** `AnsiHandler` holds the end-of-stream tail state (`$segments`, `$ss3Buffered`, `$ss3Intermediate`) as `private`/`protected`. `StreamingInspector` is a **separate** class and cannot legally touch them — reaching in (`$this->handler->segments[] = ...`, or calling the `protected getSs3Intermediate()`) throws a fatal `\Error` at runtime, e.g. a dangling SS3 (`"\x1bO"` with no final byte) at end-of-stream. Fix: give `AnsiHandler` a **public** seam instead of exposing raw state. We added `AnsiHandler::finishPending(State $stateBeforeFlush)` which encapsulates the whole end-of-stream flush (buffered text → trailing bare ESC → dangling SS3) and is now shared by both the one-shot `parse()` and the streaming `finish()`, so the two paths stay byte-for-byte identical. Note `AnsiHandler` is `final`, so `protected` on it was meaningless anyway — removed the dead `getSs3Intermediate()`/`isSs3Buffered()` accessors.
- **[pattern:spark:one-shot-vs-streaming-parity]** `Inspector::parse()` (one-shot) and `StreamingInspector::feed()`/`finish()` (chunked) must produce the same segment stream for the same bytes. When touching either, prefer routing shared logic through one method on `AnsiHandler` (see `finishPending()`) rather than duplicating the tail handling — the streaming tests (`testStreamedC0IsIsolated`, `testStreamedApcKittyGraphics`, `testStreamedSgrUnderlineColonForm`) assert this parity by diffing against `Inspector::parse()`.
